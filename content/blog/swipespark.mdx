## Stack

- **React Native + Expo** — cross-platform without compromise
- **Reanimated 3 + Gesture Handler** — worklet-based swipe at 60fps
- **Zustand + AsyncStorage** — lightweight, persisted state
- **expo-media-library** — direct, permission-scoped camera roll access
- **Swift / Kotlin** — native modules for on-device AI photo analysis

## Gesture Architecture

Achieving a true Tinder-like feel requires zero frame drops. The moment JS thread becomes a bottleneck, the experience falls apart. Reanimated solves this by running swipe translation and card rotation entirely on the UI thread inside native worklets — the JS thread is never in the critical path.

```tsx
const translateX = useSharedValue(0)

const gesture = Gesture.Pan()
  .onUpdate((e) => {
    translateX.value = e.translationX
  })
  .onEnd((e) => {
    const shouldDelete = e.translationX < -SWIPE_THRESHOLD
    translateX.value = withSpring(shouldDelete ? -SCREEN_WIDTH : 0)
  })
```

<MdxCallout type="info">
Always use `.value` to read SharedValues inside worklets — reading them outside a worklet context returns a stale snapshot and the gesture will behave inconsistently.
</MdxCallout>

## On-Device AI

SwipeSpark can automatically categorize your photos — people, animals, landscapes, indoor scenes — without sending a single byte to a server. I wrote a native module in Swift (iOS) and Kotlin (Android), bridged to React Native, that runs Vision framework inference entirely on-device. Completely offline, completely private.

## Privacy Model

No accounts. No cloud sync. No analytics. SwipeSpark accesses the device media library through the standard Expo permissions API and never transmits photo data anywhere. Everything stays on your device.

## What I Learned

- **Expo Router** has matured into a genuinely production-ready framework. Tab navigation, deep linking, and modal sheets all work exactly as you'd expect on both platforms.
- **App Store review** is slower than Play Store by roughly 2–3x. Plan accordingly.
- **Native modules** are less painful than their reputation suggests — the main cost is setting up the bridge; the actual native code is straightforward.
